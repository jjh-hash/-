# 小程序订阅消息配置说明

> 用于订单状态、支付成功、配送进度等「服务通知」推送。用户需在支付成功等时机勾选订阅后，才会收到消息。

---

## 一、公众平台配置

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → 进入你的小程序。
2. **功能** → **订阅消息** → **公共模板库**。
3. 搜索并添加与「订单」相关的模板，例如：
   - **订单状态通知**（推荐）：通常包含 订单号、订单状态、备注 等字段（thing1 / thing2 / thing3）。
4. 添加后进入 **我的模板**，复制该模板的 **模板 ID**（一串英文数字）。

---

## 二、平台配置中写入模板 ID

在云开发 **数据库** → **platform_config** 集合中，给当前使用的那条配置文档增加字段（字段名≤32 字）：

| 字段名 | 类型 | 值 |
|--------|------|-----|
| `subscribeMsgOrderTplId` | String | 上一步复制的模板 ID |

若 `platform_config` 只有一条记录，直接编辑该条，新增上述字段并保存即可。

**注意：** 当前已按公众平台模板「订单状态变更通知」配置，发送时使用的字段为：

- `thing1`：商家名称（来自店铺名称，最多 20 字）
- `thing2`：订单状态（支付成功 / 商家制作中 / 待骑手取餐 / 骑手已接单 / 配送中 / 订单已送达）
- `thing7`：温馨提示（固定「请留意订单进度」）
- `thing10`：商品名称（订单内商品名前两项拼接，最多 20 字）
- `phone_number12`：商家电话（来自店铺电话，无则填 `-`）

请确保选用的模板 ID 写入 platform_config 的 **subscribeMsgOrderTplId** 字段（若控制台字段名长度限制为 32，则用此短字段名；代码兼容旧字段名 subscribeMessageOrderStatusTemplateId）。

---

## 三、发送时机（已实现）

| 时机 | 说明 |
|------|------|
| 支付成功 | 单笔订单支付、统一付款支付成功后 |
| 商家制作中 | 商家将订单状态改为「制作中」 |
| 待骑手取餐 | 商家将订单状态改为「待取餐/已出餐」 |
| 骑手已接单 | 骑手/校园兼职在配送大厅接单 |
| 配送中 | 骑手确认取餐，订单进入配送中 |
| 订单已送达 | 骑手确认送达，订单完成 |

以上均由 **orderManage** 云函数在对应操作成功后调用 `sendOrderSubscribeMessage` 发送（未配置模板 ID 时静默跳过）。

---

## 四、用户端请求订阅（已实现）

- **结算页**：点击「提交订单」时仅请求「订单状态变更」模板（微信限制：一次多模板只展示第一个，故退款、评价分散到各自场景）。
- **申请退款**：在订单详情/订单列表点击「申请退款」时请求「退款进度通知」模板。
- **评价**：在订单详情点击「评价」时请求「订单评价提醒」模板。**注意**：用户需先点一次「评价」并勾选允许，后续订单完成时才能收到评价提醒；首单完成时若尚未点过评价，则不会收到。
- **去支付**：点击「去支付」时请求「订单状态变更」模板。

用户勾选并同意后，后续才会收到对应时机的服务通知；未勾选则不会推送。**注意**：用户若曾勾选「总是保持以上选择，不再询问」，后续同模板不会弹窗。

## 五、退款进度通知（已实现）

- **公众平台**：在「订阅消息」→「公共模板库」中搜索并添加「退款进度通知」模板，复制模板 ID。
- **数据库**：在 **platform_config** 中增加字段 `subscribeMsgRefundTplId`（或 `subscribeMessageRefundTemplateId`）。
- **发送时机**：① 用户点击「取消订单」时（取消=退款），立即发送「退款成功」；② 骑手未取餐前用户申请退款时，系统自动通过并立即发送「退款成功」；③ 骑手取餐后商家拒绝退款时，立即发送「您的餐已送出，拒绝退款」。
- **当前使用的模板字段**（退款进度通知）：
  - `thing1`：商家名称
  - `thing7`：商品名称（订单内商品名前两项拼接）
  - `amount5`：退款金额（元，字符串）
  - `thing4`：退款原因
  - `thing8`：温馨提示（如「退款成功」/「非常抱歉，你的退款申请被拒绝了」）

## 六、订单评价提醒（已实现）

- **公众平台**：在「订阅消息」→「公共模板库」中搜索并添加「订单评价提醒」模板，复制模板 ID。
- **数据库**：在 **platform_config** 中增加字段 `subscribeMsgReviewTplId`（或 `subscribeMessageReviewTemplateId`）。
- **发送时机**：订单状态变为「已完成」时（骑手确认送达、或用户确认完成），立即发送评价提醒，点击跳转评价页。
- **当前使用的模板字段**（订单评价提醒）：
  - `thing8`：商家名称
  - `amount6`：消费金额（订单金额，元）
  - `date3`：下单时间
  - `thing5`：点评提醒（如「亲，订单已完成，快来点评一下吧」）

结算页仅请求「订单状态」；退款、评价模板在用户点击「申请退款」「评价」时分别请求。

---

## 七、商家新订单通知（已实现）

- **公众平台**：在「订阅消息」→「公共模板库」中搜索并添加「新订单通知」模板（商户提醒类目），复制模板 ID。
- **数据库**：在 **platform_config** 中增加字段 `subscribeMsgNewOrderTplId`（或 `subscribeMessageNewOrderTemplateId`），值填模板 ID（如 `2pRqEGhEtt2OLGuZo4rINRvRx35fR6IVjmEK5HjM27M`）。
- **发送时机**：用户支付成功后，立即向该订单所属店铺的商家（merchants 中 storeId 匹配且 status 为 active/pending、有 openid）发送新订单通知。
- **当前使用的模板字段**（新订单通知）：
  - `thing3`：菜品名称（订单商品名前 3 项拼接，最多 20 字）
  - `amount4`：支付金额（如「99元」）
  - `time2`：要餐时间（下单时间 + 30 分钟，格式「yyyy年mm月dd日 hh:mm」）
  - `time6`：下单时间（格式「yyyy年mm月dd日 hh:mm」）
  - `thing5`：订单备注（无则填「无」，最多 20 字）
- **商家订阅**：商家需在「订单管理」页点击「开启新订单提醒」并勾选允许，后续有新订单支付成功时才会收到推送。点击消息可跳转至商家订单页。

---

### 必须完成的配置（否则弹窗只有「订单状态」、退款/评价收不到消息）

1. 打开 **云开发控制台 → 数据库 → platform_config**，编辑当前使用的那一条记录。
2. 在该记录中**新增字段**（若没有则添加，字段名可任选其一）：
   - **退款进度通知**：字段名 `subscribeMsgRefundTplId` 或 `subscribeMessageRefundTemplateId`，值填「退款进度通知」模板 ID。
   - **订单评价提醒**：字段名 `subscribeMsgReviewTplId` 或 `subscribeMessageReviewTemplateId`，值填「订单评价提醒」模板 ID。
   - **商家新订单通知**：字段名 `subscribeMsgNewOrderTplId` 或 `subscribeMessageNewOrderTemplateId`，值填「新订单通知」模板 ID（如 `2pRqEGhEtt2OLGuZo4rINRvRx35fR6IVjmEK5HjM27M`）。
3. 保存后，**重新打开结算页**（或重新进入小程序再试）。结算页提交订单时弹窗为「订单状态」；申请退款、点击评价时分别弹「退款进度」「订单评价提醒」。勾选「允许」后，后续才会收到对应服务通知。
4. **重要**：platform_config 中的模板 ID 必须来自 **微信公众平台 → 订阅消息 → 我的模板**。若从别处复制或模板未添加，会报 20001「No template data return」。

---

## 七、排查

- **报错 20001「No template data return, verify the template id exist」**：模板 ID 无效或未添加到小程序。请到 **微信公众平台 → 订阅消息 → 公共模板库** 搜索并添加对应模板，再在 **我的模板** 中复制正确的模板 ID 到 platform_config。
- **多模板同弹窗只显示一个**：微信限制，一次请求多个模板时仅展示第一个。本方案已改为分散到各场景分别请求。
- **申请退款/评价时无弹窗**：检查 platform_config 是否已配置 `subscribeMsgRefundTplId`、`subscribeMsgReviewTplId`，并确保进入过结算页或订单页以预拉模板 ID。
- **收不到退款/评价消息**：① 确认 platform_config 中已填对应模板 ID；② **关键**：用户必须在结算页弹窗中勾选「退款进度通知」「订单评价提醒」并点「允许」，未勾选则云函数会报 43101，消息不会推送；③ 评价提醒在骑手点击「送达」或用户确认完成时发送，可在云函数日志中搜索「评价提醒」查看是否发送及错误码。
- **云函数报错 43101**：用户未订阅或已拒绝该模板，需在结算页弹窗中勾选对应项并允许。
- **requestSubscribeMessage: fail last call has not ended**：上一次订阅弹窗未关闭时又触发了新的请求，需等待当前弹窗关闭后再操作；避免快速连续点击多个会触发订阅的按钮。
- **云函数报错 40037**：模板 ID 错误或未关联到本小程序，请到公众平台「我的模板」中核对。
