## 用户相关集合

### 1. users - 用户信息表

**用途：** 存储小程序用户的基本信息

**字段说明：**

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `_id` | String | 是 | 用户ID（自动生成） | `"user_123456"` |
| `openid` | String | 是 | 微信用户OpenID（唯一标识） | `"oABC123..."` |
| `unionid` | String | 否 | 微信UnionID | `"union_123..."` |
| `nickname` | String | 否 | 用户昵称 | `"张三"` |
| `avatar` | String | 否 | 用户头像URL | `"https://..."` |
| `phone` | String | 否 | 联系电话 | `"13800138000"` |
| `email` | String | 否 | 邮箱地址 | `"user@example.com"` |
| `campus` | String | 否 | 所在校区 | `"南校区"` |
| `college` | String | 否 | 所在学院 | `"计算机学院"` |
| `major` | String | 否 | 专业 | `"软件工程"` |
| `role` | String | 否 | 用户角色（user/merchant/admin） | `"user"` |
| `merchantName` | String | 否 | 商家名称（当role为merchant时） | `"我的店铺"` |
| `status` | String | 否 | 用户状态（active/banned） | `"active"` |
| `createdAt` | Date | 是 | 创建时间 | `2025-01-01T00:00:00Z` |
| `updatedAt` | Date | 是 | 更新时间 | `2025-01-01T00:00:00Z` |
| `lastLoginAt` | Date | 否 | 最后登录时间 | `2025-01-01T00:00:00Z` |

**索引建议：**
- `openid` - 唯一索引
- `status` + `createdAt` - 复合索引

## 订单相关集合

### 5. orders - 订单信息表

**用途：** 存储所有订单的详细信息（外卖订单、跑腿订单、代打订单等）

**字段说明：**

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `_id` | String | 是 | 订单ID（自动生成） | `"order_123456"` |
| `orderNo` | String | 是 | 订单号（唯一） | `"ORD1234567890ABC"` |
| `userId` | String | 是 | 用户ID（关联users._id） | `"user_123456"` |
| `userOpenid` | String | 是 | 用户OpenID（冗余字段） | `"oABC123..."` |
| `storeId` | String | 是 | 店铺ID（关联stores._id） | `"store_123456"` |
| `storeName` | String | 是 | 店铺名称（冗余字段） | `"美味餐厅"` |
| `items` | Array | 是 | 订单商品列表 | `[{productId, productName, price, quantity, spec}]` |
| `amountGoods` | Number | 是 | 商品金额（分） | `5000`（50元） |
| `amountDelivery` | Number | 是 | 配送费（分） | `300`（3元） |
| `platformFee` | Number | 是 | 平台服务费（分） | `400`（4元） |
| `platformFeeRate` | Number | 是 | 平台服务费比例 | `0.08`（8%） |
| `amountDiscount` | Number | 否 | 优惠金额（分） | `0` |
| `amountTotal` | Number | 是 | 订单总金额（分） | `5300`（53元） |
| `amountPayable` | Number | 是 | 应付金额（分） | `5300` |
| `merchantIncome` | Number | 是 | 商家收入（分） | `4600`（46元） |
| `address` | Object | 是 | 配送地址信息 | `{name, phone, address, addressDetail}` |
| `remark` | String | 否 | 订单备注 | `"不要辣"` |
| `needCutlery` | Boolean | 否 | 是否需要餐具 | `true` |
| `cutleryQuantity` | Number | 否 | 餐具数量 | `2` |
| `deliveryType` | String | 是 | 配送方式（delivery/pickup） | `"delivery"` |
| `orderStatus` | String | 是 | 订单状态 | `"pending"` |
| `payStatus` | String | 是 | 支付状态（paid/unpaid） | `"paid"` |
| `expiredAt` | Date | 是 | 订单超时时间 | `2025-01-01T00:20:00Z` |
| `hasReview` | Boolean | 否 | 是否已评价 | `false` |
| `reviewId` | String | 否 | 评价ID（关联reviews._id） | `"review_123456"` |
| `createdAt` | Date | 是 | 创建时间 | `2025-01-01T00:00:00Z` |
| `updatedAt` | Date | 是 | 更新时间 | `2025-01-01T00:00:00Z` |

**订单状态说明：**
- `pending` - 待确认（商家未接单）
- `confirmed` - 已确认（商家已接单）
- `preparing` - 制作中
- `ready` - 待配送（商家已准备好）
- `delivering` - 配送中
- `completed` - 已完成
- `cancelled` - 已取消

**索引建议：**
- `orderNo` - 唯一索引
- `userId` + `createdAt` - 复合索引
- `storeId` + `orderStatus` + `createdAt` - 复合索引
- `orderStatus` + `createdAt` - 复合索引

## 🔑 重要字段说明

### 金额字段
- **金额统一以"分"为单位存储**（如：25元存储为2500）
- 订单相关金额字段：`amountGoods`、`amountDelivery`、`platformFee`、`amountTotal`等
- 商品价格：`price`（分）

### 时间字段
- 所有集合都有 `createdAt` 和 `updatedAt` 字段
- 使用 `db.serverDate()` 创建，自动使用服务器时间
- 日期类型为 `Date` 对象

### 状态字段
- 大多数集合都有 `status` 字段，表示记录的状态
- 常见状态值：`active`、`pending`、`deleted`、`inactive` 等

### ID字段
- `_id`：所有集合的主键，自动生成
- 关联字段：使用其他集合的 `_id` 作为外键
- `openid`：微信用户唯一标识，在多个集合中使用

## 📝 注意事项

1. **数据一致性**
   - 删除商家时，需要同时处理关联的店铺、商品等数据
   - 删除订单时，需要考虑退款、评价等关联数据

2. **索引优化**
   - 根据查询频率创建合适的索引
   - 避免创建过多索引影响写入性能

3. **数据安全**
   - 密码字段使用SHA256加密存储
   - 敏感信息不要记录在日志中

4. **数据清理**
   - 定期清理过期的日志数据
   - 清理已删除的记录（软删除改为硬删除）
