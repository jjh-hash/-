# 校园外卖系统 - 云函数权限文档

> 本文档说明每个云函数需要的数据库权限和云函数调用权限  
> 最后更新：2026-04-12

---

## 📋 说明

在微信云开发中，每个云函数都需要配置相应的数据库权限。本文档列出每个云函数需要的权限配置。

**云函数访问数据库的机制：** 云函数在云端运行时，默认拥有**云环境管理员权限**，可读写该环境下已存在的任意集合，无需在控制台为「某个云函数」单独勾选「对某集合的权限」。因此「为某云函数配置对某集合的读写权限」实际只需：**在云开发控制台创建该集合**（若尚未创建），云函数即可访问。

### Web 管理端（admin-web）

- 浏览器端使用 **`@cloudbase/js-sdk`**：需开启云开发 **匿名登录**，并将部署域名加入 **Web 安全域名**。
- 调用方式与小程序管理端一致：云函数入参中带 **`adminSessionToken`**（由 `admin-web/src/cloud.js` 的 `callAdminFunction` 自动合并到请求体），各云函数的鉴权规则与小程序携带会话时相同。
- **轮播图**：Web 端通过 SDK **`uploadFile`** 将图片传到云存储（建议路径前缀 `banners/`），得到 **fileID**（`cloud://...`）后调用 `bannerManage` 的 `create`/`update`，不新增云函数。

## 支付相关

### paymentManage - 支付管理

**数据库权限：**
- `orders` 集合：
  - 读取权限：仅限订单所属用户（通过 `userOpenid` 字段验证）
  - 写入权限：无（支付状态更新由 `orderManage` 云函数处理）
- `unified_payments` 集合：读取权限（用于统一付款预下单，校验 openid 与 status）

**云函数调用权限：**
- 所有已登录用户可调用
- 需要验证订单归属权（订单的 `userOpenid` 必须与当前用户 `OPENID` 一致）

**操作说明：**
- `unifiedOrder`: 需要读取订单信息，验证订单归属和支付状态
- `unifiedPrepay`: 统一付款预下单，读取 `unified_payments` 文档（unifiedPaymentId），校验 userOpenid 与 status，调微信统一下单后返回支付参数
- `unifiedOrderDeposit`: 保证金统一下单（不读 DB，由 campusPartnerManage 调用，需传入 openid、totalFee、outTradeNo、description）
- `queryOrder`: 需要读取订单信息，验证订单归属
- `queryOrderByOutTradeNo`: 按商户订单号查询支付结果（不读 DB，用于保证金支付校验）
- `refundOrder`: 订单取消/退款（由 orderManage、refundManage 调用），支持可选 refundAmount（元）用于部分退款，需商户证书
- `refundByOutTradeNo`: 按商户订单号退款（用于保证金原路退，需商户证书）
- `transferToBalance`: 企业付款到零钱（商家提现打款），需商户证书，由 withdrawManage 调用

**配置步骤：**
1. 在云开发控制台 -> 数据库 -> 权限设置
2. 找到 `orders` 集合
3. 设置权限为：仅创建者可读写（或自定义权限，确保用户只能访问自己的订单）

## 订单相关

### orderManage - 订单管理

**数据库权限：**
- `orders` 集合：读写权限
- `stores` 集合：读取权限
- `merchants` 集合：读取权限
- `users` 集合：读取权限
- `platform_config` 集合：读取权限
- `campus_partners` 集合：读取权限（接单/抢单前校验校园兼职是否已开通）
- `unified_payments` 集合：读写权限（统一付款：创建预支付单、支付成功后更新状态）

**云函数调用权限：**
- 所有已登录用户可调用
- 部分操作需要验证权限（如商家操作需要验证商家身份；任务接单、配送抢单需校园兼职已开通或骑手审核通过）
- **管理类**：`getAdminOrderList` 需有效 `adminSessionToken`；`completeOrder` 仅允许管理员会话；`cancelOrder` 在用户/商家无权时，可凭管理员会话通过

**操作说明（统一付款）：**
- `createUnifiedOrders`: 入参 address、stores（多店 cart 数据）；为每店创建一笔订单（payStatus: unpaid，unifiedPaymentId 指向新创建的 unified_payments 文档），并创建一条 unified_payments（out_trade_no、orderIds、totalFeeFen、status: pending）；返回 orderIds、totalFeeFen、unifiedPaymentId、out_trade_no
- `updateUnifiedPaymentPaid`: 入参 unifiedPaymentId；校验 userOpenid 后将该条 unified_payments 置为 paid，并将 orderIds 中全部订单更新为 payStatus: 'paid'
- `createOrder` / `createUnifiedOrders` 金额规则：后端按地址楼层计算配送费（1-3楼=1.5元、4-6楼=2元、无法识别默认2元），线上仅收配送费（`amountPayable`）；`amountGoods` 仅用于展示代买金额；骑手收入按楼层计（1-3楼=1元，4-6楼=1.3元）
- 2026-04-06 结构优化：楼层/配送费/骑手收入规则已抽离到 `orderManage/pricingRules.js`，仅代码结构优化，不涉及权限变更
- `getDeliverOrders`：骑手配送列表返回增加 `customerName`、`customerPhone`（从订单 `address` 提取）用于联系顾客；仅返回字段补充，不涉及新增集合权限

## 用户相关

### loginUser - 用户登录

**入参说明（客户端）：**
- `code`：必填，`wx.login` 换取的登录凭证。
- `campus`：可选，`白沙校区` 或 `金水校区`（及兼容 `baisha` / `jinshui`）。传入且合法时，会写入或更新当前用户在 `users` 表中的 `campus` 字段（与首页 `homeCurrentCampus` 枚举一致）。

**返回 `data` 补充（与多端入口）：**
- `hasMerchantPortal`：布尔，`merchants` 集合中存在 `openid` 为当前用户则 `true`。
- `hasRiderPortal`：布尔，`riders` 集合中存在该 `openid`，或 `orders` 中存在 `riderOpenid` 为该 `openid` 则 `true`（与管理端用户详情骑手判断一致）。客户端在登录+选校流程结束后可据此弹出「用户端 / 商家端 / 骑手端」选择。

**数据库权限：**
- `users` 集合：读写权限
- `merchants`、`riders` 集合：**读取**（仅检测当前 `openid` 是否存在记录，用于返回 `hasMerchantPortal` / `hasRiderPortal`）
- `orders` 集合：**读取**（在无 `riders` 文档时检测 `riderOpenid`，与骑手身份兜底逻辑一致）

**云函数调用权限：**
- 所有用户可调用（包括未登录用户）

### merchantLogin - 商家登录

**入参说明：**
- `loginType`：`account`（账号密码）或 `wx`（微信一键登录）
- 账号密码登录：`account`、`password`
- 微信登录：可选传 `merchantId`。同一 `openid` 下有多条商家时，首次不传 `merchantId` 返回 **code `201`** 及 `merchants` 列表供前端选择；用户选定后再次调用并传入 `merchantId` 完成登录

**返回码：**
- `200`：登录成功
- `201`：微信登录且关联多个商家，需选择账号（`data.merchants`、`data.user`）
- `403` / `404`：非商家、用户或商家不存在等

**数据库权限：**
- `merchants` 集合：读取、更新（更新 `lastLoginAt`）
- `stores` 集合：读取权限
- `users` 集合：读取权限（微信登录校验角色）

**云函数调用权限：**
- 所有用户可调用

---

### merchantManage - 商家管理（管理端）

**数据库权限：**
- `merchants` 集合：读写权限
- `stores` 集合：读写权限
- `products`、`categories`、`store_photos`、`reviews`、`orders` 集合：删除权限（`action: delete` 时按店铺级联硬删，与「删除该商家」文案一致）
- `users` 集合：删除权限（仅当该微信下已无任何 `merchants` 记录时，删除 `role: merchant` 的用户行）
- `admin_accounts`、`admin_sessions` 集合：读取权限（校验会话）

**云函数调用权限：**
- 所有 action 需有效 `adminSessionToken`

**删除商家（delete）行为说明：**
- 汇总 `merchants.storeId` 与 `stores.merchantId === 商家_id` 的全部店铺，逐店删除子数据后再删店铺文档，避免历史重复建店残留。
- 若同一 `openid` 仍有其他商家账号，不删 `users`，避免误删多店微信号。

### getStoreList - 获取店铺列表

**数据库权限：**
- `merchants` 集合：读取权限（分页拉取全部 `status: active`，汇总 `storeId`）
- `stores` 集合：读取权限（`businessStatus: open` + 校区/分类/关键词）
- `platform_config` 集合：读取权限（配送费展示，可选）
- `products` 集合：读取权限（按店统计在售商品数，count）

**云函数调用权限：**
- 所有用户可调用（包括未登录用户）

**说明：** 已对 `merchants` 显式 `limit(1000)` 拉取 active 商家，避免单次 `get` 默认条数上限导致部分已审核商家未参与列表（≥1000 时需后续建索引并分页）；`storeId` 较多时对 `_id in` 做分片 `or` 组合，避免 `in` 数组超长。

- `merchants` 集合：读取权限（用于筛选已审核商家的店铺）

**云函数调用权限：**

**说明：** 返回跨店铺的商品列表；`merchants` 同样 `limit(1000)`，店铺与商品条件中的 `in` 与 `getStoreList` 一致做分片。支持分页、关键词与 `categoryName`（与 `categories.name` 匹配）。

### campusPartnerManage - 校园兼职管理

**数据库权限：**
- `campus_partners` 集合：读写权限（按 openid 查询/新增/更新本人记录）
- `platform_config` 集合：读取权限（获取 depositAmount）
- `deposit_orders` 集合：读写权限（创建保证金支付单、更新支付/退款状态）

**云函数调用权限：**
- 已登录用户可调用（需 openid）
- 会调用 `paymentManage` 的 unifiedOrderDeposit、queryOrderByOutTradeNo、refundByOutTradeNo

**说明：** getStatus、apply、createDepositPayment（创建支付单并返回调起支付参数）、confirmDeposit（传入 depositOrderId 校验支付后开通）、requestRefund（缴纳满 30 天后申请并立即执行原路退款）。保证金金额来自 platform_config；锁定期为缴纳满 30 天可退。

**配置步骤（deposit_orders）：**
1. 打开 [微信公众平台](https://mp.weixin.qq.com/) → 进入你的小程序 → **开发** → **开发管理** → **云开发**，进入云开发控制台。
2. 左侧选择 **数据库**，在集合列表中点击 **「+」新建集合**。
3. 集合名称填写：`deposit_orders`，确定创建。
4. 无需再为 campusPartnerManage 单独配置权限：云函数默认具备管理员权限，创建好 `deposit_orders` 后，campusPartnerManage 即可正常读写该集合。

- 用户端：所有已登录用户可调用（需要验证订单归属）
- 商家端：商家用户可调用（需要验证店铺归属）
- 管理端：`getRefundDetail` 跨用户查看、`updateRefundStatus` 等需有效管理员会话（或兼容 `admins` 集合 openid）

**操作说明：**
- `createRefund`：骑手未取餐前（pending/confirmed/preparing/ready）用户申请退款时自动通过，调用 paymentManage.refundOrder 执行退款并立即发送订阅消息；骑手取餐后不可申请
- `updateRefundStatus`：商家同意/拒绝时，若拒绝且订单已配送，发送「您的餐已送出，拒绝退款」
- `sendRefundNotificationForCancel`：由 orderManage.cancelOrder 调用，取消订单=退款时发送「退款成功」订阅消息

## 📝 权限配置建议

### 数据库权限设置原则

1. **最小权限原则**：只授予必要的权限
2. **用户数据隔离**：用户只能访问自己的数据
3. **管理员权限**：管理员可以访问所有数据
4. **商家权限**：商家只能访问自己店铺的数据

### 常见权限配置

- **仅创建者可读写**：适用于用户个人数据（地址、订单等）
- **所有用户可读，仅创建者可写**：适用于公开数据（商品、店铺等）
- **仅管理员可读写**：适用于系统配置、管理数据

## ⚠️ 注意事项

1. **安全验证**：即使配置了数据库权限，云函数内部仍需进行权限验证
2. **敏感信息**：支付相关配置（API密钥等）应存储在云函数中，不要暴露给前端
3. **定期审查**：定期检查云函数权限配置，确保符合安全要求
