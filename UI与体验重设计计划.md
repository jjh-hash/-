## 二、重设计目标与原则

### 2.1 目标

- **统一视觉语言**：建立可复用的设计系统（色彩、字体、圆角、阴影、间距），消除各端「各做各的」观感。
- **提升可读与操作效率**：信息层级清晰、主次分明，关键操作（下单、支付、接单）更易发现、点击区域合理。
- **强化校园场景认同**：在保持通用性的前提下，通过插画、文案、图标风格体现「校园外卖/校园生活」属性，与纯商业外卖区分。
- **保持可实施性**：不推翻现有架构，以样式与组件优化为主，必要时补充或替换部分 wxml 结构，便于分阶段落地。

### 2.2 原则

- **一致性优先**：同类型元素（按钮、卡片、列表项、空状态）在同一端内风格统一，跨端可共用基础组件与变量。
- **渐进增强**：先做设计系统与核心页面（首页、个人中心、订单流、店铺/结算），再覆盖任务大厅、商家端、骑手端、管理端。
- **适配与安全区**：所有固定底栏、悬浮按钮考虑 `env(safe-area-inset-bottom)`，避免与刘海/横条重叠。
- **性能与包体**：控制大图与动效使用，优先用 CSS 与本地小图标，避免首屏过重。

## 三、设计系统（Design System）

### 3.1 色彩

建议在 `app.wxss` 或全局样式文件中用 CSS 变量统一管理，便于后续主题或多端微调：

| 用途 | 建议变量名 | 建议值 | 说明 |
|------|------------|--------|------|
| 主色/品牌色 | `--color-primary` | 如 `#07c160`（微信绿）或 `#1677ff`（蓝） | 主按钮、选中态、链接 |
| 主色浅 | `--color-primary-light` | 对应 10%～15% 透明度 | 背景、标签 |
| 成功 | `--color-success` | 如 `#07c160` | 成功态、完成 |
| 警告 | `--color-warning` | 如 `#fa8c16` | 待支付、待确认 |
| 危险 | `--color-danger` | 如 `#ff4d4f` | 取消、退款、异常 |
| 正文 | `--color-text` | 如 `#1f2937` | 主要文字 |
| 次要文字 | `--color-text-secondary` | 如 `#6b7280` | 辅助说明 |
| 占位/禁用 | `--color-text-placeholder` | 如 `#9ca3af` | 占位、禁用 |
| 背景页 | `--color-bg-page` | 如 `#f5f6f8` | 页面底 |
| 背景卡片 | `--color-bg-card` | `#ffffff` | 卡片、浮层 |
| 分割线 | `--color-border` | 如 `#eef0f3` | 列表分割、边框 |

### 3.2 字体与字号

- **字体**：系统默认（如 `-apple-system, BlinkMacSystemFont, 'PingFang SC', 'Helvetica Neue'`），保证可读性。
- **字号层级建议**：
  - 大标题：18–20px、字重 600
  - 标题/卡片标题：16–17px、字重 600
  - 正文：14–15px、字重 400
  - 辅助/说明：12–13px、字重 400
  - 小字/标签：11px、字重 400

### 3.3 圆角与阴影

- **圆角**：卡片 12px，按钮 8px，标签/胶囊 20px（或 999px），头像 50%。
- **阴影**：卡片轻阴影 `0 1px 3px rgba(0,0,0,0.06)`，悬浮可略强 `0 2px 8px rgba(0,0,0,0.08)`；避免过重导致「脏」。

### 3.4 间距

- **页面边距**：左右 12–16px 统一。
- **模块间距**：16–24px。
- **列表项内边距**：12–16px。
- **底部栏**：`padding-bottom` 含 `env(safe-area-inset-bottom)`。

### 3.5 组件规范（可抽成公共 class 或 component）

- **主按钮**：背景主色、白字、圆角 8px、高度约 44px、无边框。
- **次按钮**：白底/浅灰底、主色字或深灰字、细边框。
- **卡片**：白底、统一圆角与内边距、可选轻阴影。
- **列表项**：左图/左文 + 右箭头或右文案，高度建议 ≥ 44px，可点击区域充足。
- **空状态**：插画或图标 + 主文案 + 可选副文案 + 可选操作按钮，居中排版。
- **标签**：小字号、圆角胶囊、主色/灰底，与正文有对比。

## 四、页面与模块优化规划

### 4.1 用户端（C 端）— 优先

| 页面/模块 | 现状问题（示例） | 优化方向 |
|-----------|------------------|----------|
| **首页** | 导航栏、筛选栏、店铺卡片、底部栏风格可能不统一 | 统一导航栏高度与状态栏适配；筛选栏（综合排序/销量/星级）保持胶囊样式并统一主色；店铺卡片统一圆角、阴影、间距与主次信息层级；底部栏已组件化，统一图标与选中色 |
| **个人中心** | 头像区、三块卡片、菜单项样式可能零散 | 头像区与昵称/ID 层级清晰；三块卡片视觉统一（图标、标题、副文案、箭头）；菜单项统一高度与点击态；联系客服等入口样式与其它菜单一致 |
| **订单列表** | 分类标签、订单卡片、空状态可能简陋 | 分类标签与设计系统统一；订单卡片信息分组清晰（店铺/时间/状态/金额/操作）；空状态插画+文案+「去逛逛」等引导 |
| **订单详情** | 状态条、信息块、操作按钮可能杂乱 | 状态进度或状态条统一风格；地址/商品/金额分块清晰；主操作（支付/确认收货）突出，次操作弱化 |
| **任务大厅** | 筛选、任务卡片、空状态 | 与订单列表在「列表卡片」上风格统一；筛选栏与首页筛选栏视觉语言一致；空状态友好引导 |
| **店铺列表/店铺详情** | 店铺卡片、商品列表、加购按钮 | 店铺头图与信息层级；商品卡片统一；加购与结算栏固定底、安全区 |
| **结算页** | 地址、商品、配送时间、支付方式、提交订单 | 模块分块清晰；必填项与可选项区分；提交按钮固定底、主色、足够大 |
| **分类页（代拿/游戏/悬赏）** | 表单与发布按钮 | 表单项标签与输入框统一；发布按钮主色、固定底或明显位置 |
| **我的信息/地址/关于** | 表单项、列表项 | 与设计系统表单项、列表项一致 |

### 4.2 商家端

| 页面/模块 | 优化方向 |
|-----------|----------|
| 登录/注册 | 与 C 端登录风格可区分但统一设计语言；错误提示、加载态明确 |
| 工作台 | 核心数据（今日订单、收入）突出；快捷入口图标与文案统一 |
| 订单管理 | 订单列表与 C 端订单卡片风格可复用；状态筛选与操作按钮清晰 |
| 商品/类目/评价/设置 | 列表与表单遵循设计系统；设置项分组与说明清晰 |

### 4.3 骑手端

| 页面/模块 | 优化方向 |
|-----------|----------|
| 首页（待抢单/待取货/待送达） | 订单卡片信息层级清晰；抢单/导航等主操作突出；空状态友好 |
| 个人中心/设置/收入/统计 | 与 C 端个人中心、列表与卡片风格统一 |

### 4.4 管理员端

| 页面/模块 | 优化方向 |
|-----------|----------|
| 数据概况 | 核心指标卡片化、图表简洁可读 |
| 管理系统/列表/详情 | 表格或列表可读；操作按钮与状态颜色统一；与商家端列表风格可共用基础组件 |

## 五、体验增强点（非纯视觉）

- **加载态**：列表加载用骨架屏或统一 loading 样式，避免空白闪屏。
- **错误与空状态**：网络错误、无数据、无权限等均有统一模板（图标+文案+操作），避免纯白或系统默认提示。
- **反馈**：关键操作（下单、支付、接单）有明确 Toast 或结果页，失败时有可操作提示。
- **可点击区域**：列表项、按钮等保证最小点击高度（如 44px），避免误触或难点。

| **Phase 1** | 设计系统落地：全局色彩/字体/圆角/间距变量，基础按钮/卡片/列表项 class 或组件 | `app.wxss` 变量 + 可选 `components/design-tokens` 或公共 wxss |
