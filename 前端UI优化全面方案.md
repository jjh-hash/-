# 校园外卖 - 前端 UI 优化全面方案

> 高级 UI 优化设计师视角 | 基于现有设计系统与页面结构  
> 编制日期：2026-02-17

---

## 一、现状分析

### 1.1 已有基础

| 维度 | 现状 | 评价 |
|------|------|------|
| 设计系统 | app.wxss 已定义 `--color-*`、`--radius-*`、`--shadow-*`、`--space-*` 及 ds-* 工具类 | 基础完善，可在此基础上扩展 |
| 组件库 | 未引入第三方 UI 库，全部使用原生组件 | 包体小，但需自建交互与动效 |
| 页面覆盖 | 用户端、商家端、骑手端、管理端、二手均已接入设计变量 | 色彩与圆角统一，但视觉层次与细节仍有提升空间 |
| 体验增强 | 骨架屏、空状态、加载态、错误态类已提供 | 部分页面已接入，可推广至更多场景 |

### 1.2 主要问题

1. **图标与视觉符号**：首页搜索用 emoji（🔍）、快捷分类用 emoji、店铺公告用 emoji（📢），观感不统一且难以控制样式。
2. **信息层级**：部分卡片标题与正文对比不足，缺少明确的视觉权重区分。
3. **交互反馈**：按钮、列表项点击态多为简单 opacity，缺少微动效与状态过渡。
4. **留白与节奏**：部分页面模块间距不统一，列表项密度偏高。
5. **导航与 Tab**：店铺详情左右分栏、订单分类 Tab 等可增强选中态与过渡动画。
6. **表单与输入**：输入框、选择器、开关等样式分散，未形成统一表单组件规范。

## 二、设计原则与目标

### 2.1 原则

- **克制**：不堆砌装饰，以信息清晰与操作效率为先。
- **一致**：同类型元素（按钮、卡片、列表、表单）全项目统一。
- **可维护**：优先扩展设计系统变量与公共组件，减少单页硬编码。
- **兼容**：适配不同机型安全区、深色模式（可选）、大字号（可选）。

### 2.2 目标

- 提升首屏与核心流程的视觉舒适度与专业感。
- 强化主操作（下单、支付、接单）的可见性与点击信心。
- 减少用户认知负担，信息层级一目了然。
- 在不大幅增加包体的前提下，适度引入动效与微交互。

## 三、组件库选型建议

### 3.1 方案对比

| 方案 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| **纯原生 + 自研** | 包体最小，完全可控 | 开发成本高，需自建 Toast、弹窗、Picker 等 | 当前方案，可继续深化 |
| **Vant Weapp** | 组件丰富、文档完善、社区活跃 | 包体增加约 200KB+，样式需覆盖以匹配设计系统 | 快速补齐表单、弹窗、反馈类组件 |
| **WeUI for 小程序** | 官方风格、轻量 | 组件较少，偏基础 | 仅需基础组件时 |
| **TDesign 小程序** | 腾讯设计语言、组件完整 | 包体较大，风格需适配 | 大型项目、多端统一 |

### 3.2 推荐策略

**阶段一（当前）**：继续以**原生 + 设计系统**为主，通过扩展 app.wxss 与公共组件补齐短板，不引入第三方库，保持包体与维护成本可控。

**阶段二（可选）**：若需快速补齐 Toast、ActionSheet、Picker、Dialog、Loading 等，可引入 **Vant Weapp** 并按设计变量覆盖主题色与圆角，仅按需分包引用。

## 四、设计系统增强

### 4.1 新增变量建议（app.wxss）

```css
/* 现有基础上补充 */
--color-text-tertiary: #9ca3af;        /* 更弱辅助文字 */
--radius-input: 8px;                   /* 输入框圆角 */
--radius-badge: 4px;                   /* 小徽章圆角 */
--shadow-float: 0 4px 16px rgba(0,0,0,0.08);  /* 浮层/弹窗阴影 */
--transition-fast: 0.15s ease;          /* 快速过渡 */
--transition-normal: 0.25s ease;       /* 常规过渡 */
```

### 4.2 新增工具类建议

| 类名 | 用途 |
|------|------|
| `.ds-input` | 统一输入框样式（高度、圆角、边框、内边距） |
| `.ds-divider` | 分割线（水平线，可带左右留白） |
| `.ds-badge` | 数字/红点徽章（角标、通知数） |
| `.ds-avatar-sm/md/lg` | 头像尺寸规范（32/48/64px） |
| `.ds-press` | 通用点击态（opacity 或 scale） |
| `.ds-fade-in` | 淡入动画（列表项、弹窗） |

### 4.3 图标方案

- **替换 emoji**：使用 iconfont 或本地 SVG/PNG 图标，统一尺寸与颜色（可复用 `--color-text-placeholder`、`--color-primary`）。
- **推荐**：在 `images/icons/` 或 `components/icon/` 下维护一套 24x24、32x32 的 PNG/SVG，或接入 iconfont 小程序方案。
- **临时**：若暂不引入图标库，可将 emoji 替换为文字符号（如「搜」「公告」）并配合背景色块，减少视觉噪音。

## 五、页面级优化规划

### 5.1 用户端（C 端）— 高优先级

| 页面 | 优化项 | 具体措施 |
|------|--------|----------|
| **首页** | 搜索框 | 替换 emoji 为图标或文字符号；增加 focus 态边框高亮 |
| | 快捷分类 | 替换 emoji 为图标；统一圆形容器尺寸与阴影；增加点击态 |
| | 轮播图 | 指示器样式统一（圆点/条形）；可增加渐变遮罩提升文字可读性 |
| | 店铺卡片 | 左侧封面增加圆角与占位图；右侧信息行距与字号层级优化；增加店铺标签（如「热销」「新店」） |
| **个人中心** | 头像区 | 头像增加边框或光晕；昵称与副信息层级区分 |
| | 菜单卡片 | 图标统一尺寸与颜色（可用 filter 或换图标资源）；增加分组标题（可选）；箭头与右侧信息对齐 |
| **订单列表** | 分类 Tab | 选中态下划线或背景更明显；增加滑动过渡 |
| | 订单卡片 | 状态标签与金额视觉权重；操作按钮（再来一单、评价）样式统一 |
| | 空状态 | 使用 ds-empty 模板；插画或图标 + 引导文案 + 按钮 |
| **订单详情** | 状态条 | 进度式或步骤式展示；主色与灰色区分 |
| | 信息块 | 地址、商品、金额分块清晰；分割线统一 |
| | 操作区 | 主按钮（支付/确认收货）突出；次按钮弱化 |
| **任务大厅** | 任务卡片 | 与订单卡片风格统一；类型标签、时间、金额层级清晰 |
| | 筛选栏 | 与首页筛选胶囊风格一致 |
| **店铺详情** | 店铺头 | 公告区图标替换；滚动公告样式优化 |
| | 分类侧栏 | 选中态背景与左边框；过渡动画 |
| | 商品卡片 | 图片圆角统一；价格与加购按钮层级；数量选择器样式统一 |
| | 购物车/结算栏 | 固定底栏阴影与安全区；角标样式统一 |
| **结算页** | 地址卡片 | 图标 + 信息层级；编辑入口明显 |
| | 商品列表 | 缩略图与信息对齐；金额右对齐 |
| | 提交按钮 | 主色、全宽、固定底、安全区 |

### 5.2 商家端 / 骑手端 / 管理端

- 与用户端保持**同一设计语言**：按钮、卡片、列表项、表单输入样式一致。
- 数据类页面（工作台、统计、仪表盘）：数字突出、图表简洁、卡片留白适中。
- 管理端列表：表头与行高统一；操作按钮（通过/拒绝）使用语义色。

### 5.3 二手模块

- 列表卡片：与店铺卡片在布局上可区分，但圆角、阴影、间距遵循设计系统。
- 发布页：表单输入、图片上传、发布按钮与结算页风格统一。

## 六、动效与微交互

### 6.1 建议引入的动效

| 场景 | 动效 | 实现方式 |
|------|------|----------|
| 列表项点击 | 短暂 opacity 或 scale(0.98) | CSS transition + :active |
| 弹窗出现 | 淡入 + 轻微上移 | CSS animation 或 wx.createAnimation |
| 切换 Tab | 下划线滑动或背景滑动 | transition + transform |
| 骨架屏 | 渐变闪烁 | 已有 ds-skeleton |
| 加载更多 | 底部 spinner | 已有 ds-loading-spinner |
| 成功/失败反馈 | Toast 淡入淡出 | wx.showToast 或自定义 Toast 组件 |

### 6.2 原则

- 动效时长控制在 0.15s–0.3s，避免拖沓。
- 减少全屏或大面积动画，以局部反馈为主。
- 考虑低端机性能，避免复杂 CSS 动画叠加。

## 七、实施阶段建议

### Phase 1：设计系统扩展（1–2 天）

- 在 app.wxss 中补充变量与工具类（ds-input、ds-divider、ds-badge、ds-press 等）。
- 确定图标方案：建立 `images/icons/` 或 `components/icon`，替换首页、店铺详情等处的 emoji。

### Phase 2：用户端核心页面（3–5 天）

- 首页：搜索、快捷分类、轮播、店铺卡片。
- 个人中心：头像区、菜单卡片。
- 订单列表：Tab、订单卡片、空状态。
- 订单详情：状态条、信息块、操作区。

### Phase 3：交易流程页面（2–3 天）

- 店铺详情：头图、分类侧栏、商品卡片、购物车栏。
- 结算页：地址、商品、配送、支付、提交按钮。

### Phase 4：任务大厅与二手（1–2 天）

- 任务大厅：筛选、任务卡片、空状态。
- 二手列表/详情/发布：与主流程风格统一。

### Phase 5：商家端 / 骑手端 / 管理端（2–3 天）

- 按设计系统查漏补缺；数据类页面数字与图表优化。

## 八、可复用的微信原生组件

以下为微信小程序原生组件，可直接使用并配合设计变量样式覆盖：

| 组件 | 用途 | 样式要点 |
|------|------|----------|
| `button` | 主/次按钮 | open-type 支持分享、客服等；可设 plain、size |
| `input` | 单行输入 | placeholder-class、focus 态 |
| `textarea` | 多行输入 | 固定高度或 auto-height |
| `picker` | 选择器 | mode=selector/multiSelector/date/time/region |
| `switch` | 开关 | color 属性可设主色 |
| `slider` | 滑块 | activeColor、backgroundColor |
| `progress` | 进度条 | stroke-width、color |
| `swiper` | 轮播 | indicator-dots、indicator-color、indicator-active-color |
| `scroll-view` | 滚动区域 | scroll-x/scroll-y、enhanced、show-scrollbar |
| `movable-area` + `movable-view` | 可拖动 | 可用于抽屉、悬浮球等 |

## 十、总结

本方案在现有设计系统基础上，通过**扩展变量与工具类**、**替换 emoji 为图标**、**统一表单与列表样式**、**适度增加动效**，提升全项目 UI 的一致性与专业感。优先实施用户端核心页面与交易流程，再覆盖其余端。不强制引入第三方组件库，以原生 + 自研为主，保持包体与维护成本可控；若后续需要快速补齐 Toast、Picker、Dialog 等，可再评估引入 Vant Weapp 并按设计变量做主题覆盖。

若认可本方案，可从 **Phase 1（设计系统扩展 + 图标方案）** 开始落地；需要从某一具体页面或模块着手时，可直接指定。
