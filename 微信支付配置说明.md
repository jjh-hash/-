# 微信支付配置说明

> 本文档说明如何配置微信支付功能  
> 最后更新：2025-01-03

---

## 📋 配置步骤

### 1. 获取微信支付商户号

1. 登录微信支付商户平台：https://pay.weixin.qq.com
2. 在"账户中心" -> "商户信息"中获取：
   - **商户号（mchId）**
   - **API密钥（apiKey）**

### 2. 配置平台配置

在云数据库的 `platform_config` 集合中添加微信支付配置：

```javascript
{
  // ... 其他配置字段
  wechatPay: {
    mchId: "你的商户号",      // 例如：1234567890
    apiKey: "你的API密钥"     // 32位字符串
  }
}
```

**配置方法：**

1. 通过云开发控制台 -> 数据库 -> `platform_config` 集合
2. 找到或创建配置记录
3. 添加 `wechatPay` 字段，包含 `mchId` 和 `apiKey`

### 3. 配置支付回调（可选）

支付回调用于接收微信支付的异步通知，确保订单状态及时更新。

**配置步骤：**

1. 在微信支付商户平台配置支付回调URL
2. 创建HTTP触发云函数处理支付回调
3. 回调URL格式：`https://你的云环境域名/paymentNotify`

**注意：** 如果不配置回调，系统会在支付成功后主动更新订单状态，但建议配置回调以确保可靠性。

## 📝 支付流程

### 用户支付流程

1. **创建订单**
   - 用户在结算页面提交订单
   - 系统创建订单，状态为 `unpaid`

2. **统一下单**
   - 调用 `paymentManage` 云函数的 `unifiedOrder` 接口
   - 返回支付参数（timeStamp, nonceStr, package, signType, paySign）

3. **调起支付**
   - 使用 `wx.requestPayment` 调起微信支付
   - 用户完成支付

4. **更新订单状态**
   - 支付成功后，更新订单状态为 `paid`
   - 跳转到订单页面

### 支付回调流程（可选）

1. 微信支付服务器发送支付结果到回调URL
2. HTTP触发云函数接收回调数据
3. 验证签名和订单信息
4. 更新订单状态为 `paid`

---

## ⚠️ 注意事项

1. **安全性**
   - API密钥必须保密，不要泄露
   - 支付回调需要验证签名，防止伪造请求

2. **测试环境**
   - 开发环境可以使用微信支付沙箱进行测试
   - 沙箱环境需要单独申请和配置

3. **金额单位**
   - 所有金额以"分"为单位（1元 = 100分）
   - 统一下单时传入的 `totalFee` 必须是整数（分）

4. **订单号**
   - 订单号（out_trade_no）必须唯一
   - 建议使用订单号格式：`ORD + 时间戳 + 随机数`

5. **支付超时**
   - 微信支付订单默认30分钟超时
   - 超时后订单自动取消，需要重新下单

## 🔍 常见问题

### 1. 统一下单失败

**可能原因：**
- 商户号或API密钥配置错误
- 订单金额格式错误（必须是整数，单位：分）
- 订单号重复

**解决方法：**
- 检查平台配置中的 `wechatPay.mchId` 和 `wechatPay.apiKey`
- 确保金额以分为单位，且为整数
- 确保订单号唯一

### 2. 支付成功但订单状态未更新

**可能原因：**
- 支付回调未配置或配置错误
- 支付成功后的主动更新失败

**解决方法：**
- 检查支付回调配置
- 查看云函数日志，确认更新订单状态的操作是否成功
- 可以手动调用 `orderManage` 的 `updateOrderPayStatus` 接口更新状态

### 3. 支付参数错误

**可能原因：**
- 签名生成错误
- 支付参数格式不正确

**解决方法：**
- 检查签名生成逻辑
- 确保所有支付参数格式正确（timeStamp为字符串，package包含prepay_id等）
