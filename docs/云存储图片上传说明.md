# 静态图片上传云存储说明

主包体积超限（8916KB > 2048KB）主要因 `images/` 下大图未压缩：
- `express-icon.png` 2.4MB
- `gaming-icon.png` 3.0MB  
- `secondhand-icon.png` 2.9MB
- `service-flash-bg.png` 88KB
- `delivery-boy-icon.png` 96KB

**合计约 8.4MB**，上传至云存储后可显著减小主包体积。

## 上传步骤

1. 打开**微信开发者工具**，进入项目
2. 点击顶部 **云开发**，进入云开发控制台
3. 左侧选择 **云存储**
4. 新建文件夹 `static`（若不存在）
5. 进入 `static`，点击 **上传文件**
6. 选择本地 `images/` 目录下 5 个文件上传：
   - express-icon.png
   - gaming-icon.png
   - secondhand-icon.png
   - service-flash-bg.png
   - delivery-boy-icon.png
7. 上传完成后，点击每个文件右侧 **详情**，复制 **File ID**（格式如 `cloud://cloud1-xxx.xxx/static/express-icon.png`）
8. 打开 `config/cloudImages.js`，将复制的 File ID 填入对应字段：
   ```js
   const config = {
     expressIcon: 'cloud://你复制的fileID',
     gamingIcon: 'cloud://你复制的fileID',
     secondhandIcon: 'cloud://你复制的fileID',
     serviceFlashBg: 'cloud://你复制的fileID',
     deliveryBoyIcon: 'cloud://你复制的fileID'
   };
   ```
9. 保存后重新编译，主包将不再包含这些大图（`project.config.json` 已配置 `packOptions.ignore` 排除 `images/`）

## ⚠️ 重要

**必须先完成上传并填入 fileID，再编译/上传**。因 `images/` 已从打包中排除，若未配置云 URL，图片将 404。

## 注意事项

- 云存储 fileID 可直接用于 `<image src="{{fileID}}">`，小程序会自动处理
- 首次打开可能需加载网络图，建议保留骨架或占位
