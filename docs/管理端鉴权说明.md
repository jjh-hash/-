# 管理端鉴权现状说明

## 一、结论概览

| 项目 | 现状 |
|------|------|
| **非管理员是否能看到管理员入口** | **不能**。仅当本地存在 `adminToken`（已用管理员账号登录过且未退出）时才显示「管理端」。 |
| **非管理员是否能进入管理端** | **能**。若通过其它方式进入（如直接扫管理端页码），数据概况页仍无鉴权，建议首页补鉴权。 |
| **管理端是否做鉴权** | **部分做了**。入口已鉴权（按 adminToken 显隐）；首页未鉴权；部分子页有鉴权。 |

---

## 二、入口与首页

1. **管理员入口（已做入口鉴权）**
   - 位置：`pages/profile/index.js` → 「更多服务」中的「管理端」。
   - 展示逻辑：**仅当本地存在 `adminToken` 时**才显示「管理端」入口（在 `onLoad` / `onShow` 中调用 `buildMenuSections()` 根据 `wx.getStorageSync('adminToken')` 动态生成菜单）。无管理员权限的用户看不到该入口。
   - 点击后：`wx.reLaunch({ url: '/pages/admin-dashboard/index' })`。建议在管理端首页再做一次鉴权，未登录时打回登录页。

2. **管理端首页（数据概况）**
   - 主包：`pages/admin-dashboard/index.js`
   - 分包：`subpackages/admin/pages/admin-dashboard/index.js`
   - **onLoad / onShow**：仅加载数据，**无 `verifyAdminAccess()` 或 `adminToken` 校验**。
   - 结果：非管理员进入后可以正常看到数据概况页。

3. **管理功能页**
   - `pages/admin-management/index.js` 与 `subpackages/admin/pages/admin-management/index.js`
   - **onLoad / onShow**：无鉴权。
   - 结果：从数据概况通过底部 Tab 或「管理功能」进入后，非管理员也能看到管理功能菜单。

---

## 三、有鉴权的管理端子页

以下页面在 **onLoad** 中调用了 **verifyAdminAccess()**（或等价逻辑），会检查 `adminToken`，无则弹窗并 reLaunch 到商家注册页（管理员登录页）：

- `admin-merchant-list`（商家管理）
- `admin-order-list`（订单管理）
- `admin-user-list`（用户管理）
- `admin-rider-list`（骑手设置）
- `admin-finance`（财务管理）
- `admin-statistics`（数据统计）
- `admin-log-list`（操作日志）
- `admin-batch-audit`（批量审核）
- `admin-change-password`（修改密码，使用 `AdminAuth.checkAdminAuth()`）

---

## 四、未做鉴权的管理端页面

以下页面 **onLoad 未做** `adminToken` / 管理员身份校验，理论上从管理功能菜单或其它入口跳转过去后，非管理员也能打开（若路由允许）：

- `admin-dashboard`（数据概况）※ 首页
- `admin-management`（管理功能）※ 菜单页
- `admin-profile`（个人中心）
- `admin-settings`（系统设置）
- `admin-order-detail`（订单详情）
- `admin-refund-detail`（退款详情）
- `admin-merchant-detail`（商家详情）
- `admin-user-detail`（用户详情）
- `admin-banner-edit`（轮播图编辑）
- `admin-announcement`（公告管理）

（实际是否可进入还受小程序分包与导航路径限制，但页面自身未做鉴权。）

---

## 五、建议改动（若需收紧权限）

1. **管理端首页鉴权**  
   在 **admin-dashboard**（主包与分包两处）的 **onLoad** 中增加与其它子页一致的 `verifyAdminAccess()`（或统一封装的鉴权方法）。无 `adminToken` 时弹窗提示并 reLaunch 到管理员登录页（如商家注册页），避免非管理员看到数据概况。

2. **管理功能页鉴权**  
   在 **admin-management** 的 **onLoad** 中同样增加鉴权，避免未登录管理员看到管理功能菜单。

3. **个人中心「管理端」入口（可选）**  
   - 方案 A：保持所有人可见，依赖上述首页/管理功能鉴权，未登录管理员点进去会被打回登录页。  
   - 方案 B：仅管理员可见。需在个人中心根据 `adminToken`（或服务端返回的角色）决定是否展示「管理端」菜单项；若当前无管理员角色接口，可先采用方案 A。

4. **其余管理端子页**  
   对 **admin-profile、admin-settings、admin-order-detail、admin-refund-detail、admin-merchant-detail、admin-user-detail、admin-banner-edit、admin-announcement** 等在 **onLoad** 中补充鉴权，与现有「有鉴权」子页保持一致。

---

## 六、云函数侧

- 订单、退款等云函数内已有「是否管理员」校验（如查 `admins` 集合），接口层有权限控制。
- 小程序端鉴权主要用于：控制谁能看到管理端入口、谁能打开管理端页面，避免未授权用户看到管理端界面和敏感数据。
