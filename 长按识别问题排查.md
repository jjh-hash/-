# 长按识别二维码问题排查

## 问题说明
如果长按识别功能不生效，请按以下步骤排查：

## 1. 确保图片路径正确

### 本地图片路径
- 图片必须放在项目的 `images` 目录下
- 文件名：`wechat-pay-qrcode.png`
- 完整路径：`/images/wechat-pay-qrcode.png`

### 网络图片
- 如果使用网络图片，系统会自动下载到本地
- 确保网络连接正常
- 图片URL必须是有效的HTTP/HTTPS链接

## 2. 检查图片是否加载成功

打开微信开发者工具的调试器，查看 Console 输出：
- 如果看到 "二维码下载成功，本地路径: ..." 说明图片已下载
- 如果看到 "二维码图片加载成功" 说明图片已显示
- 如果看到错误信息，请检查图片路径

## 3. 长按识别功能要求

### 必要条件
1. **图片必须是本地路径**（临时文件路径也可以）
2. **图片必须已加载完成**
3. **需要在真机上测试**（模拟器可能不支持）

### 测试方法
1. 在真机上运行小程序
2. 打开收款码弹窗
3. 长按二维码图片（按住1-2秒）
4. 应该会弹出菜单，包含"识别图中二维码"选项

## 4. 常见问题

### 问题1：长按没有反应
**原因**：图片可能是网络路径，未下载到本地
**解决**：检查代码中的 `downloadQRCode` 方法是否正常执行

### 问题2：图片不显示
**原因**：图片路径错误或图片不存在
**解决**：
- 检查图片文件是否存在
- 检查路径是否正确
- 查看控制台错误信息

### 问题3：模拟器上无法长按识别
**原因**：模拟器不支持长按识别功能
**解决**：必须在真机上测试

## 5. 调试步骤

1. **检查图片路径**
   ```javascript
   // 在 generateQRCode 方法中添加日志
   console.log('收款码URL:', qrcodeUrl);
   console.log('图片路径:', this.data.qrcodeImage);
   ```

2. **检查图片加载**
   ```javascript
   // 在 onImageLoad 方法中确认图片已加载
   console.log('图片加载成功');
   ```

3. **真机测试**
   - 使用微信开发者工具的真机调试功能
   - 或编译后上传到微信，在手机上测试

## 6. 替代方案

如果长按识别仍然不生效，可以考虑：

1. **使用预览图片功能**
   ```javascript
   wx.previewImage({
     urls: [this.data.qrcodeImage],
     current: this.data.qrcodeImage
   });
   ```
   在预览模式下，用户可以长按识别

2. **添加"识别二维码"按钮**
   - 点击按钮后调用 `wx.scanCode` API
   - 允许用户从相册选择二维码图片

## 7. 代码检查清单

- [ ] `show-menu-by-longpress="{{true}}"` 属性已添加
- [ ] 图片路径是本地路径（不是网络URL）
- [ ] 图片已成功加载（查看 onImageLoad 日志）
- [ ] 在真机上测试（不是模拟器）
- [ ] 图片文件存在且路径正确
