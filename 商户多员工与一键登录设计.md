# 商户多员工与一键登录设计（待完成）

> 本文档记录「一个店铺多员工 + openid 强对应 + 一键登录 / 账号密码登录」的设计方案，**当前标记为待完成，暂不实现**。

---

## 1. 目标

- **商户与 openid 强对应**：每个商家账号（每条 `merchants` 记录）可绑定一个微信 openid，绑定后该微信可一键登录。
- **一个店铺多人**：同一店铺下可有多个员工（owner + staff），每人一条 `merchants` 记录，`storeId` 相同。
- **登录方式**：
  - 店铺所有员工：支持**一键登录**（绑 openid 后）+ **账号密码登录**。
  - 管理员：在商户登录页仅使用**账号密码登录**，用于对账等，不依赖 openid。

---

## 2. 数据与关系（沿用现有结构）

- **stores**：店铺表，一个店一条记录。
- **merchants**：商家账号表，一条记录 = 一个可登录的人；多人 = 多条记录，同一 `storeId`，`role` 为 owner / staff。
- **openid**：每个商家账号可绑一个 openid；建议 `merchants.openid` 唯一索引（一个 openid 只绑一个商家账号）。

无需新增表，只需在业务上明确「merchant = 人，store = 店，一个 store 多个人 = 多条 merchant 同 storeId」。

---

## 3. 员工如何“绑定该店铺”

以下方式任选或组合，最终都是：为该店铺新增一条 `merchants` 记录（`storeId` = 该店，`role` = staff）。

### 3.1 方式 A：店长在后台添加员工（推荐先做）

- 店长/管理员在后台「添加员工」：输入员工信息，系统为该店铺新增一条 `merchants` 记录（storeId、role=staff、account、初始密码，openid 先空）。
- 把账号与初始密码发给员工；员工在商户登录页用**账号密码**登录一次，登录后可在「我的」或设置里点击「绑定当前微信」，云函数将当前 openid 写入该 merchant 记录，之后即可一键登录。

### 3.2 方式 B：邀请码 / 邀请链接

- 店长生成店铺邀请码或带 storeId 的邀请链接（或小程序码）。
- 员工打开小程序进入「加入店铺」页，输入邀请码或通过链接带入 storeId；云函数校验后为当前 openid 创建一条该 storeId 的 merchant 记录（可自动生成 account/password），并直接写入 openid，即可一键登录。

### 3.3 方式 C：员工申请 + 店长审核

- 员工在小程序内申请成为某店铺员工，店长在后台审核；通过时创建该 storeId 的 merchant 记录并可选写入 openid。

## 4. 前端与接口（待实现时补充）

- **商户登录页**：保留账号密码登录；增加「微信一键登录」按钮（调 wx.login → 云函数用 openid 查 merchants → 有则登录，无则提示用账号密码或联系店长）。
- **一键登录云函数**：code → openid → 查 `merchants`（openid + status）→ 返回登录态与 storeId。
- **账号密码登录**：保持现有逻辑；管理员对账即用商户账号密码登录，无需单独入口。
- **绑定当前微信**：账号密码登录成功后，可选将当前 openid 写入本 merchant 记录（或设置页「绑定当前微信」）。
- **后台**：店长「添加员工」页/接口（创建 merchant、storeId、role=staff、account、password）；可选邀请码生成与校验接口。
